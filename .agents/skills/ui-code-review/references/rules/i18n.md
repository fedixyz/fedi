# i18n: no hardcoded user-facing strings

Every string a user can see must go through i18n, never be hardcoded.

- covers screen copy, button labels, toast and overlay text, and error messages surfaced to the UI or to miniapp callers
- use `t()` with a key defined in `ui/common/localization/en/common.json`
- a hardcoded user-facing English string is a defect, not a nit: the app ships in ~20 locales, so raw English is broken UI for most users
- applies to new code, not only regressions
